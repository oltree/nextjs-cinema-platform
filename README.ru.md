# Full Online Cinema — Frontend

> **[Read in English](README.md)**

Полнофункциональное веб-приложение онлайн-кинотеатра, созданное на **Next.js 13**, **React 18** и **TypeScript**. Платформа поддерживает авторизацию пользователей, просмотр и воспроизведение фильмов, оценки и избранное, навигацию по жанрам и актёрам, а также полноценную административную панель для управления контентом.

## Ключевые технические решения

### JWT-авторизация с автообновлением токена

Axios-инстанс (`src/api/config.ts`) прикрепляет access-токен к каждому запросу через request interceptor. Response interceptor перехватывает ошибки `401` — конкретно "jwt expired" и "jwt must be provided" — автоматически вызывает эндпоинт обновления токена и повторяет оригинальный запрос. При неудаче обновления токены удаляются из cookies и пользователь разлогинивается.

```
Запрос   →  [прикрепить access token]  →  API
Ответ    ←  [401 jwt expired?]         ←  API
               ↓ да
          getNewTokens()
               ↓
          повтор оригинального запроса
```

### Ролевой контроль доступа

Каждая Next.js-страница может экспортировать флаг `isOnlyAdmin` или `isOnlyUser`. `AuthProvider` читает эти флаги, проверяет состояние пользователя в Redux и выполняет редирект — без дополнительного бойлерплейта на уровне страниц.

### Статическая генерация для главной страницы

Главная страница использует `getStaticProps` для предварительного рендеринга трендовых фильмов и топ-актёров во время сборки, обеспечивая мгновенную загрузку без обращения к API при каждом посещении.

### Кастомный видеоплеер

Плеер (`src/components/ui/video-player/`) построен на нативном HTML5-элементе `<video>`. Реализует:
- Переключение воспроизведения/паузы
- Перемотку вперёд/назад на 10 секунд
- Fullscreen API
- Прогресс-бар с отображением текущего времени
- Авторизационный шлюз: неавторизованным пользователям вместо плеера показывается приглашение войти

### Архитектура административной панели

Маршруты админ-панели находятся под `/manage/`. Для каждой сущности (фильм, жанр, актёр, пользователь) существует страница-список с поиском/сортировкой и страница редактирования с формами. Загрузка файлов идёт через `FileService`, который отправляет `multipart/form-data` на бэкенд и получает URL в ответ. Динамические импорты (`next/dynamic`) используются для WYSIWYG и тяжёлых компонентов, чтобы не раздувать пользовательский бандл.

### Тёмная кинотеатральная тема Tailwind

Дизайн использует кастомную тёмную тему из `tailwind.config.js`:
- **Основной акцент:** `#E30B13` (кинотеатральный красный)
- **Золото рейтинга:** `#F5C521` (жёлтый в стиле IMDb)
- **Шкала фонов:** `gray-950` → `gray-800` (почти чёрные тёмные серые)
- Кастомные утилиты: `.air-block` (тёмная округлая карточка), `.btn-primary`, `.text-link`

## Запуск проекта

### Требования

- Node.js 18+
- npm 9+ или yarn
- Запущенный бэкенд по адресу `http://localhost:4200` ([репозиторий бэкенда](https://github.com/oltree/full-online-cinema-be))

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/oltree/Full-online-cinema-fe.git
cd Full-online-cinema-fe

# Установить зависимости
npm install

# Скопировать файл окружения и заполнить значения
cp .env.example .env.local
```

### Разработка

```bash
npm run dev
# Откроется на http://localhost:3000
```

### Продакшн-сборка

```bash
npm run build
npm run start
```

## Переменные окружения

| Переменная | Пример | Описание |
|---|---|---|
| `NEXT_PUBLIC_APP_URL` | `http://localhost:3000` | Публичный URL фронтенда |
| `NEXT_PUBLIC_SERVER_URL` | `http://localhost:4200` | URL бэкенд API-сервера |
| `NEXT_PUBLIC_APP_ENV` | `development` | Окружение (`development` / `production`) |

Rewrites в `next.config.js` автоматически проксируют `/api/*` и `/uploads/*` на бэкенд, поэтому настройка CORS в разработке не требуется.
